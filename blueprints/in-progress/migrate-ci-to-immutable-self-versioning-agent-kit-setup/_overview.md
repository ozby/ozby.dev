# Migrate CI to immutable self-versioning agent-kit setup

**Goal:** Replace duplicated consumer-owned agent-kit package pins with the setup-owned immutable GitHub Action contract, while retaining explicit compatibility coverage for agent-kit 3.1.10 and making governance checks future-version agnostic.

## Architecture Overview

```text
setup-vp (immutable SHA) -> setup-wp (immutable SHA) -> action resolves its own agent-kit version
                                      |
                                      +-> consumer workflows use `wp` without package pins
```

## Key Decisions

| Decision | Choice | Rationale |
| --- | --- | --- |
| Migration owner | Run agent-kit PR #509 setup source | Consumer output must be generated by the owner implementation, not hand-patched |
| Version policy | Self-versioning setup action | Version bumps must not require edits across consumer workflows |
| Governance | Discover every workflow and reject legacy patterns | New workflows cannot silently reintroduce package pins |

## Non-goals

- Hand-patching behavior that belongs in the setup owner
- Updating unrelated generated agent surfaces

## Risks

| Risk | Impact | Mitigation |
| --- | --- | --- |

| New workflow bypasses contract | Legacy pins return | Governance enumerates all workflow YAML files |
